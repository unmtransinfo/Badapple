For BADAPPLE promiscuity analysis. Utilizes HScaf package, accesses database containing compound and scaffold assay statistics to generate promiscuity scores.

BADAPPLE = Bio- Activity Data Associative Promiscuity Pattern Learning Engine

Functionality

  • Utilizes HScaf (edu.unm.health.biocomp.hscaf) package.
  • Accesses database containing compound and scaffold assay statistics.
  • Database server may be PostgreSQL or MySql.
  • Generates promiscuity scores for scaffolds and compounds.

Known bugs, issues

  • Scaffold analysis must be consistent at client and in database. If client dissects input query molecule into scaffolds, these must match exactly the scaffolds stored in the database. If scaffold analysis implementations comply strictly with the HScaf algorithm, different toolkits and/or versions can be used. An alternate approach is to rely on a database function for scaffold analysis. A prior implementation used ChemAxon SMARTS to store scaffolds as queries in a JChem-cartridge Oracle database.
  • Apart from the scaffold analysis is the efficient and rigorous indexing of scaffolds via canonical smiles. As with compound databases, it cannot be assumed that two implementations (toolkit or version) result in the same canonical smiles. However, the client toolkit normally does not need to canonicalize a query, which can be done via database cartridge function (e.g. in OpenChord, openbabel.cansmiles()).
